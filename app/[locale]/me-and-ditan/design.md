# 《我与地坛》设计文档

## 1. 内容理解

### 主题
史铁生的经典散文节选，讲述作者在双腿残废后与北京地坛古园的深厚缘分。通过十五年的相伴，他在这座荒芜但不衰败的古园中思考生死、寻找生命的意义。

### 情绪
- **沉静内敛**：不是热烈的情感宣泄，而是深沉的内心独白
- **苍凉而不绝望**：有忧郁，但更多是释然和坦然
- **时间的厚重感**：四百年的等待，十五年的陪伴，恒常与无常交织
- **生命的韧性**：园子荒芜但并不衰败，生命在静默中蓬勃

### 目标读者
对生命意义有思考的读者，喜欢沉浸式阅读体验的人

## 2. 设计直觉

### 第一感受
- 一座古老的园子，斑驳的墙，苍老的柏树
- 轮椅在落叶中缓缓滚动的轨迹
- 落日的余晖洒在祭坛的石阶上
- 时间仿佛凝固又仿佛流动

### 脑海中的画面
- 色彩：水墨调、灰褐色、苍绿色、暗金色（落日）、深红色（朱门）
- 形状：古柏的剪影、祭坛的轮廓、轮椅的轨迹
- 动态：缓慢、静谧，像时间本身在流动

## 3. 设计方案

### 视觉风格

**色彩系统：白日斑驳**

核心意象：午后阳光透过古柏树叶洒下的光影，明亮但不刺眼，温暖中带着时间的痕迹

- 背景：`#f5f2eb`（日晒宣纸色）- 温暖的米白
- 斑驳光：`#fff8e7`（阳光落点）- 淡金白
- 斑驳影：`#e8e2d5`（树影）- 浅灰
- 正文：`#3d3a35`（墨色）- 沉稳不刺眼
- 标题：`#2a2725`（深墨）- 庄重
- 点缀绿：`#6b7c5e`（苍柏色）- 生命感
- 点缀金：`#c9a86c`（落日余晖）- 温暖

**斑驳效果实现**
- 背景叠加淡金色渐变光斑（模拟透过树叶的阳光）
- 微妙的明暗交替纹理
- 滚动时光影轻微移动，营造"时间流动"感

**字体**
- 标题：思源宋体，沉稳庄重
- 正文：系统衬线体，阅读舒适
- 引用：楷体风格，手写感

**视觉元素**
- 古柏剪影作为装饰元素
- 轮椅轨迹作为导航/进度指示
- 淡化的祭坛轮廓作为背景
- 微妙的纸张纹理增加质感

**风格关键词**
水墨、沉静、时间感、中式、极简

### 交互设计

**1. 园中生命动画**（核心交互）

文中描写的小生命做成微动画，穿插在段落之间，让页面"活"起来：

| 生命 | 动画效果 | 出现位置 |
|------|----------|----------|
| 蜂儿 | 如一朵小雾悬停在半空，轻微上下浮动 | 段落旁边，hover 时轻微移动 |
| 蚂蚁 | 摇头晃脑捋触须，然后转身疾行而去 | 从段落边缘爬过，有路径动画 |
| 瓢虫 | 爬行一段后停下，展翅飞走 | 停在文字上，点击后飞走 |
| 蝉蜕 | 静止不动，"寂寞如一间空屋" | 固定装饰，微微透明 |
| 露水 | 在草叶上滚动、聚集、坠落、摔开金光 | 滚动触发，有粒子效果 |

**技术实现**：
- 使用 SVG + framer-motion 实现
- 动画触发方式：滚动进入视口时自动播放 / hover 触发
- 位置：随机分布在段落间隙或文字旁边
- 性能：使用 CSS transform，避免重绘

**2. 古园地图探索**（核心交互）

一个可交互的地坛俯视图，用户点击不同区域查看对应的文字片段：


| 区域 | 点击显示内容 |
|------|-------------|
| 祭坛 | "除去那座祭坛我不能上去而只能从各个角度张望它" |
| 古柏 | "那些苍黑的古柏，你忧郁的时候它们镇静地站在那儿..." |
| 殿堂 | "除去几座殿堂我无法进去" |
| 草地 | "差不多它的每一平米草地上都有过我的车轮印" |
| 颓墙 | "去它的老树下或荒草边或颓墙旁，去默坐，去呆想" |
| 轮椅轨迹 | 显示作者在园中的漫游路线动画 |

**视觉风格**：
- 简笔画/水墨风格的俯视图
- 淡雅的色彩，与页面整体风格一致
- 当前选中区域高亮，其他区域淡化
- 点击后文字以浮层或侧边栏形式展示

**3. 四季轮回器**
文章末尾"譬如"段落的六个意象可以做成一个沉浸式的四季/场景切换组件：
- 祭坛石门中的落日
- 雨燕高歌
- 冬天雪地上孩子的脚印
- 苍黑的古柏
- 暴雨骤临
- 秋风落叶

用户可以切换不同场景，每个场景有对应的视觉和文字。

**4. 时间线**（可选）
- 展示"四百年前地坛建成 → 作者出生 → 双腿残废 → 进入园中 → 十五年后"的时间脉络
- 用轮椅轨迹的视觉隐喻

**5. 阅读进度**（可选）
- 用轮椅在园中漫游的轨迹来显示阅读进度
- 随着滚动，轮椅轨迹逐渐延长

### 沉浸式设计

**核心隐喻：在古园中漫步**
- 整个页面就是一座古园
- 滚动就是在园中行走
- 不同段落是园中的不同角落

**布局**
- 垂直滚动，但每个段落有不同的"场景感"
- 适当的留白，给读者思考的空间
- 文字不要太密集，保持阅读的从容

**导航逻辑**
- 简洁的章节锚点
- 可以快速跳转到"关于死"、"关于活"等核心段落

### 响应式设计

**桌面端**
- 文字居中，最大宽度限制保证阅读舒适
- 四季轮回器横向排列
- 充足的边距和留白

**移动端**
- 全宽布局，但保持内边距
- 四季轮回器改为纵向滚动或滑动切换
- 字号适当增大，保证可读性
- 减少装饰性动画，保证性能

## 4. 技术实现

### 核心组件

**基础组件**
1. `Hero.tsx` - 封面，地坛意象
2. `HeroCompact.tsx` - 首页卡片版本
3. `Content.tsx` - 主内容容器
4. `SeasonScenes.tsx` - 四季/场景切换组件

**交互组件**（新增）
5. `GardenCreatures.tsx` - 园中生命动画容器
   - `Bee.tsx` - 蜂儿悬停动画
   - `Ant.tsx` - 蚂蚁爬行动画
   - `Ladybug.tsx` - 瓢虫展翅动画
   - `CicadaShell.tsx` - 蝉蜕静态装饰
   - `Dewdrop.tsx` - 露水滚落动画
6. `GardenMap.tsx` - 古园地图探索组件
   - 可点击的 SVG 地图
   - 区域高亮 + 文字浮层

**可选组件**
7. `Timeline.tsx` - 时间线组件
8. `ProgressTrail.tsx` - 轮椅轨迹阅读进度

### 依赖库
- `framer-motion` - 平滑动画
- `next-intl` - 多语言

### 性能考虑
- 装饰性 SVG 保持简洁
- 动画使用 CSS transform，避免重绘
- 图片（如有）使用 Next.js Image 优化
- 场景切换使用懒加载
- 生命动画使用 `useInView` 控制，视口外不渲染
- 地图使用纯 SVG，避免图片加载
