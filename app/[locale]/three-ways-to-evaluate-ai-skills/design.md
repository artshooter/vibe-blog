# 设计文档：三个角度评判AI技能的价值

## 内容理解

这是一篇方法论文章，教读者如何评估一个AI技能是否值得学习。作者提出了三个评判维度：

1. **理解本质** - 能否帮助理解AI的工作原理和边界
2. **产出成果** - 能否产出实际可用的东西
3. **提升能力** - 能否培养迁移性的通用能力

文章结构清晰，每个维度都有具体的例子（正例和反例），最后给出综合判断原则。这是一篇实用主义的思考框架文章。

## 设计直觉

**第一感受**：
- 清晰、理性、有条理
- 像一个决策工具或评估框架
- 三个维度形成稳定的"三角形"结构

**脑海画面**：
- 三个相交的圆（韦恩图）
- 或三角形的三个顶点
- 清晰的线条和标注

**阅读节奏**：
- 不需要情感渲染，需要清晰呈现
- 读者应该能快速理解框架，然后自己应用

**色彩联想**：
- 冷静的蓝色系（理性思考）
- 青绿色作为强调（成长、价值）
- 干净的白色背景

## 视觉风格

### 色彩方案

```
主色调：#1e3a5f（深蓝）- 理性、专业
辅助色：#0d9488（青绿）- 价值、成长
背景色：#f8fafc（浅灰白）
文字色：#1e293b（深灰）
边框/分隔：#e2e8f0（浅灰）
```

### 字体

- 标题：思源黑体或系统黑体，稍粗
- 正文：系统默认，舒适的行高（1.75）

### 视觉元素

- **三角形**：作为核心视觉符号，代表三个维度的平衡
- **圆点**：标记三个维度，用不同色调区分
- **连接线**：表示三者的关联性
- **卡片**：每个维度用卡片承载，边框清晰

## 交互设计

### 核心交互：AI技能评估器

在文章末尾添加一个交互组件，让读者可以实际使用这个框架：

**功能**：
1. 输入框：输入想要评估的AI技能名称
2. 三个维度评分：
   - 理解本质：是/部分/否
   - 产出成果：是/部分/否
   - 提升能力：是/部分/否
3. 实时显示综合评价（高/中/低学习价值）

**交互细节**：
- 选择时有轻微的动画反馈
- 综合评价区域根据结果变色（绿色高价值、黄色中等、灰色低价值）

### 辅助交互

- 三个维度卡片悬停时有轻微放大效果
- 例子部分（正例/反例）用不同颜色区分

## 沉浸式设计

### 核心隐喻：三脚架/三角形

三脚架之所以稳定，是因为有三个支撑点。同样，一个有价值的AI技能，最好能同时满足三个维度。

**布局体现**：
- Hero 区域展示一个抽象的三角形图形
- 三个维度部分，每个占据三角形的一个"角"
- 移动端保持垂直布局，但用视觉元素（圆点、连线）暗示三者的关联

**不强制隐喻**：
如果三角形布局太复杂，也可以用简单的三栏/三段式，只要保持清晰的结构感即可。

## 响应式考虑

### 桌面端（≥768px）

- Hero：居中的标题 + 三角形图形
- 三个维度：可以并排展示，或者用 tab 切换
- 评估器：较大的交互区域

### 移动端（<768px）

- Hero：简化的标题，图形缩小
- 三个维度：垂直堆叠，卡片式设计
- 评估器：全宽布局，按钮足够大便于点击

## 技术实现

### 组件结构

```
components/three-ways-to-evaluate-ai-skills/
├── index.tsx          # 导出 Article 对象
├── Hero.tsx           # 标题 + 三角形视觉
├── Content.tsx        # 主体内容
├── DimensionCard.tsx  # 单个维度卡片
└── SkillEvaluator.tsx # 交互式评估器
```

### 关键技术点

- 使用 Framer Motion 实现卡片动画和评估器交互
- 评估器状态用 React useState 管理
- Tailwind CSS 实现响应式布局

### 动画

- 页面加载：三个维度卡片依次淡入（stagger effect）
- 评估器选择：选项切换时的缩放动画
- 结果展示：颜色渐变过渡
